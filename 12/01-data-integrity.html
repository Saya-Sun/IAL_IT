📘 Data Integrity

Detailed Notes | Edexcel IT A2 Unit 3

🔹 Overview

📊
Definition

Data integrity refers to the reliability, accuracy and consistency of data throughout its lifecycle (collection → storage → usage → archiving).

⭐
Why It Matters
  • Ensures correct business decisions
  • Prevents misleading information
  • Supports data governance
  • Improves trust in systems

🔹 Components of Data Integrity

✔️ Accuracy

Data must be correct and error-free.

Example: Correct customer email address.
🔁 Consistency

Data follows the same format and rules.

Example: Same phone number format.
📦 Completeness

No missing values in dataset.

Example: Sales must include quantity sold.
🔒 Reliability
  • Data available when needed
  • Backups ensure recovery
  • Validation ensures correctness
🛡️ Security
  • Prevents unauthorised access
  • Protects from data corruption
  • Includes encryption & permissions

🔹 Maintaining Data Integrity

✅ Validation

Checks data before entry.

Prevents incorrect formats and errors.
💾 Backup

Regular copies of data.

Used for recovery after failure.
🔑 Access Control

Restricts access to data.

Password, permissions, encryption.
📋 Audit Trail

Tracks all data changes.

Ensures accountability.

🔹 Audit Trail Details

  • Timestamp: Date & time of action
  • User: Who performed the action
  • Action: Create, edit, delete, access
  • Data Affected: What was changed
  • Source: System or application used
