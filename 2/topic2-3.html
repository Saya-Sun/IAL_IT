🌐 Types of Network Models

A network model describes how devices are structured and how they communicate.

💡 Exam Tip: Always mention structure + control + examples
1

🖥️ Client-Server Network

Central server controls network

Structure: Clients request services from a central server.

Examples of servers:

  • File server → stores files
  • Web server → hosts websites
  • Print server → manages printing

Advantages:

  • Centralised security
  • Automatic backups
  • Easy software updates
  • Clients can switch devices easily

Disadvantages:

  • Expensive setup
  • Needs specialist staff
  • If server fails → network stops ❗
2

🔗 Peer-to-Peer Network

No central server • Equal devices

Structure: All devices (peers) share resources directly.

Advantages:

  • Cheap to set up
  • No specialist staff needed
  • Devices share files/resources
  • If one fails → others still work

Disadvantages:

  • Less secure ❗
  • No central backup
  • Performance drops with many devices
3

📡 Ad Hoc Network

Temporary network • No infrastructure

Structure: Devices connect directly for short-term use.

Example: Students sharing files in a park without Wi-Fi.

Advantages:

  • Very quick setup
  • No equipment needed
  • Cheap

Disadvantages:

  • Weak security
  • Limited size
  • Not all devices supported
4

📱 Tethering

Phone shares internet (hotspot)

Structure: Mobile phone acts as a router.

Connections:

  • Wi-Fi (most common)
  • Bluetooth
  • USB

Advantages:

  • Internet anywhere
  • No need multiple data plans
  • More secure than public Wi-Fi

Disadvantages:

  • Drains battery 🔋
  • Uses mobile data
  • Needs strong password

⚖️ Quick Comparison

Client-Server = central control
Peer-to-Peer = equal devices
Ad Hoc = temporary network
Tethering = phone hotspot
💡 Exam Tip: - Large networks → Client-Server - Small/simple → Peer-to-Peer
