🌐 Network Protocols

Protocols are rules that control how data is transmitted across a network.

💡 Exam Tip: Protocol = rules for communication between devices
📊 Types of Network Protocols
1

📚 OSI Model

7-layer theoretical framework
  • Developed in 1984
  • Divides communication into 7 layers
  • Each layer has a specific function
  • Used to understand how data travels

Layers (Top → Bottom):

  • 7. Application → user interaction
  • 6. Presentation → format/encrypt
  • 5. Session → connection control
  • 4. Transport → error checking
  • 3. Network → routing (IP)
  • 2. Data Link → MAC addressing
  • 1. Physical → cables/signals
Layer No. Layer Name Function Protocols / Examples
7 Application User interaction, network services HTTP, HTTPS, FTP, SMTP
6 Presentation Data format, encryption, compression SSL, TLS
5 Session Establishes and manages sessions NetBIOS, RPC
4 Transport Reliable data transfer, error checking TCP, UDP
3 Network Routing and addressing IP, ICMP
2 Data Link MAC addressing, frame transmission Ethernet, PPP
1 Physical Transmission of raw bits Cables, signals
2

🌍 TCP/IP Model

4-layer practical model (used on Internet)
  • Developed by US Department of Defense
  • Main protocol used on the internet
  • More practical than OSI

Layers:

  • Application → combines top 3 OSI layers
  • Transport → uses TCP
  • Internet → uses IP
  • Link → hardware + transmission
3

📞 VoIP

Voice communication over internet
  • Converts voice → digital data
  • Compresses and transmits data
  • Used in apps like Skype, Zoom
  • Cheaper than traditional calls
4

📡 SIP

Controls VoIP communication
  • Starts, manages and ends calls
  • Works with VoIP
  • Operates at application layer

⚖️ Key Differences

OSI = theoretical model (7 layers)
TCP/IP = real-world model (4 layers)
VoIP = voice over internet
SIP = manages VoIP calls
