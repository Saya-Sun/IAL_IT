📘 IGCSE IAL IT Topic 2

📍 Addressing in a Network

Devices in a network need addresses so data can be sent to the correct destination.

💡 Exam Tip: Compare MAC vs IP (VERY common question)
🔑 Types of Addresses
1

🔗 MAC Address

Physical (hardware) address of a device
  • Assigned by manufacturer
  • Stored in NIC (Network Interface Card)
  • Unique for every device ❗
  • Written in hexadecimal (0–9, A–F)
  • 12 digits grouped in pairs
  • Format: XX:XX:XX:XX:XX:XX
  • Example: 3C:5A:B4:01:63:D2

Structure:

  • First 3 pairs → Manufacturer ID
  • Last 3 pairs → Device serial number

Use in Network:

  • Used in LAN communication
  • Included in every data packet
  • Switch reads MAC address to send data to correct device
💡 Exam Tip: MAC address works at Data Link Layer (Layer 2)
2

🌐 IP Address

Logical address used to identify a device on a network
  • Assigned by router (DHCP) or ISP
  • Can be static (fixed) or dynamic (changes)
  • Used to send data between networks
  • Each device must have a unique IP in a network

Types of IP Address:

  • Static IP → does not change
  • Dynamic IP → changes when reconnecting

How it works:

  • IP address is added to every data packet
  • Router reads IP to find destination
  • Used for communication across networks (WAN/Internet)
💡 Exam Tip: IP address works at Network Layer (Layer 3)

⚖️ MAC vs IP Address

Feature MAC Address IP Address
Type Physical Logical
Assigned by Manufacturer Router / ISP
Changeable No Yes (dynamic)
Use Within LAN Across networks
💡 MAC = hardware identity, IP = network location
3

🌍 IPv4

32-bit address
  • 4 sets of numbers
  • Written in denary
  • Separated by dots
  • Example: 128.11.32.40
4

🌐 IPv6

128-bit address
  • 8 groups of hexadecimal numbers
  • Separated by colons
  • Much larger address space
  • Example: AB01:EF01:2368:BB30

📊 IPv4 vs IPv6

Feature IPv4 IPv6
Bits 32-bit 128-bit
Format Denary Hexadecimal
Structure 4 numbers 8 groups
Separator . :
Address Space Limited Very large
