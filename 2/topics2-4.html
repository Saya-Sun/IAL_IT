📊 Network Metrics

Network metrics are used to measure the performance of a network.

💡 Exam Tip: Always define + give unit/example
⚡ Key Metrics
1

⚡ Speed

How fast data is transferred
  • Measured in bps (bits per second)
  • Higher speed = faster data transfer
2

📶 Bandwidth

Maximum data capacity
  • Measured in Mbps / Gbps
  • Higher bandwidth → faster network
  • Shared between devices ❗
3

📊 Throughput

Actual data transferred
  • Real performance (not maximum)
  • Always lower than bandwidth
⚖️ Speed vs Bandwidth vs Throughput
Feature Speed Bandwidth Throughput
Meaning How fast data is transferred Maximum data capacity Actual data transferred
Type General term Theoretical maximum Real performance
Relationship Depends on bandwidth Limits speed Always ≤ bandwidth
💡 Exam Tip: Bandwidth = maximum Throughput = actual Speed = general term
4

📈 Scalability

Ability to handle growth
  • Can increase capacity when needed
  • Important for large organisations
5

⏱️ Latency

Delay in data transmission
  • Measured in milliseconds (ms)
  • Low latency = better performance
6

❌ Error Rate

Percentage of errors
  • Includes packet loss & corruption
  • High error → lower performance
7

📦 Packet Loss

Lost data packets
  • Measured as percentage
  • Even 1% affects performance ❗
8

🟢 Availability

Network uptime
  • Measured as % time available
⛔ Downtime

Downtime is when a network is unavailable and cannot be used.

Downtime reduces network availability ❗

Main Causes:

  • ⚡ Power failure → devices shut down
  • 🛠️ Hardware failure → router/server breaks
  • 💻 Software issues → bugs or crashes
  • 🌐 Network congestion → too much traffic
  • 🔄 Maintenance → updates or repairs
  • 🔓 Cyber attacks → e.g. DDoS
  • 🌧️ Environmental issues → storms, heat
9

📉 Jitter

Variation in delay
  • Caused by congestion
  • Affects streaming & calls

⚖️ Important Differences

Bandwidth = maximum speed
Throughput = actual speed
Latency = delay
💡 VERY COMMON: Bandwidth vs Throughput vs Latency
