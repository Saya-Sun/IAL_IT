🌐 Internet vs World Wide Web

📌 Key Difference

Internet: The physical infrastructure used to send data worldwide.

World Wide Web (WWW): The collection of web pages accessed using the internet.

💡 Exam Tip: Internet = infrastructure, WWW = web pages

🌍 How a Web Page is Accessed

Step-by-step
  1. User enters URL
  2. DNS converts domain → IP
  3. Browser sends request
  4. Server sends web page
💡 Exam Tip: Mention DNS + IP conversion
1

🌐 DNS (Domain Name System)

Converts domain names into IP addresses

DNS is a system that translates human-readable website names (like www.google.com) into IP addresses that computers use to locate servers.

🔄 How DNS Works (Step-by-Step)

  1. User enters a URL into the browser
  2. Browser extracts the domain name
  3. Request sent to a DNS server
  4. If not found → forwarded up the DNS hierarchy
  5. Root server helps locate correct DNS server
  6. Correct IP address is returned
  7. Browser connects to web server using IP

🏗️ DNS Hierarchy

  • Local DNS Server – first point of contact
  • Root DNS Server – top-level server
  • Authoritative DNS Server – stores actual domain records

⚡ Why DNS is Important

  • Humans remember names, not IP addresses
  • Makes the internet user-friendly
  • Enables fast access to websites
💡 Exam Tip: DNS = Domain Name → IP Address
2

🖥️ Web Server

Stores and sends pages

A web server is a computer that stores web pages and sends them to users when requested by a web browser.

  • Stores web content – holds HTML pages, images, videos and files
  • Handles requests – receives requests from web browsers and responds
  • Uses HTTP/HTTPS – communicates using standard web protocols
  • Has a unique IP address – allows it to be identified on the internet
3

🔗 URL (Uniform Resource Locator)

The address used to locate a web page or resource on the internet

A URL is what a user types into a web browser to access a specific web page. Example: https://www.pearson.com/index.html

🧩 Structure of a URL

  • Protocol: https:// → defines how data is transferred
  • Domain Name: www.pearson.com → identifies the website
  • Path: /index.html → specifies the exact page/resource

🔍 Explanation

  • Protocol (HTTP/HTTPS) – rules used for communication
  • Domain Name – human-readable name converted by DNS
  • Path/File Name – location of the resource on the server

⚡ Why URLs are Important

  • Allow users to easily access web pages
  • Provide a unique address for each resource
  • Work with DNS to locate servers
💡 Exam Tip: URL = Protocol + Domain Name + Path

✨ Features of the World Wide Web


1

🔗 Hypertext

Links between pages and resources

Hypertext allows users to click links to move between web pages, documents, or media such as videos. It makes navigation fast and interactive.

Example: Clicking a link to open another website.

2

📱 Cross Platform

Works on different devices and systems

The WWW can be accessed on many devices such as computers, smartphones, tablets, smart TVs and gaming consoles.

Benefit: Users can access the web anywhere.

3

🌍 Distributed

Data is stored across many servers worldwide

The WWW is not stored in one place. Instead, it is spread across many servers in different locations, reducing traffic and improving performance.

Benefit: Faster access and better reliability.

4

📏 Open Standards

Shared rules for communication

The WWW uses open standards such as HTML and HTTP so that different systems and devices can communicate with each other.

Managed by: Internet organisations.

5

🔄 Dynamic

Content is constantly updated

Web pages are regularly updated and can change based on user input, making the web interactive and current.

Example: Social media feeds updating in real time.

6

👥 Web 2.0

Users create and share content

Web 2.0 allows users to contribute content such as posts, comments, videos and blogs, making the web interactive and social.

  • Web 1.0: Read-only (users only view content)
  • Web 2.0: Read + write (users create content)
