🌐 Static vs Dynamic Web Pages

📌 Key Idea

Static: Content stays the same

Dynamic: Content changes when requested

💡 Exam Tip: Static = fixed, Dynamic = changes
1

📄 Static Web Pages

Same content for every user

Written using HTML, CSS and JavaScript. Content does not change unless manually updated.

  • No processing required
  • Server sends page directly
✅ Advantages
  • Easy to build
  • Low cost
  • Fast loading
  • Easy to cache
2

🔄 Dynamic Web Pages

Content changes for each user

Generated when requested using PHP, ASP or AJAX with databases.

  • Server processes data
  • Database may be used
✅ Advantages
  • Personalised content
  • Easier updates
  • Scalable

⚖️ Comparison

FeatureStaticDynamic
ContentSameChanges
ProcessingLowHigh
SpeedFastSlower
PersonalisationNoYes
💡 Exam Tip: Static = fast, Dynamic = flexible

🧠 Quick Memory

  • Static → same page
  • Dynamic → generated page
  • Static → no processing
  • Dynamic → database
