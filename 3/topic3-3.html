💻 Client-side vs Server-side Scripting

📌 Key Idea

Client-side scripting: Runs on the user's computer

Server-side scripting: Runs on the web server

💡 Exam Tip: Client = browser, Server = web server
1

💻 Client-side Scripting

Runs in the user's browser
  • Used in static web pages
  • Executed by the web browser
  • No need to contact server
  • Reduces server workload
  • Main language: JavaScript
  • Code is visible to users
2

🖥️ Server-side Scripting

Runs on the web server
  • Used in dynamic web pages
  • Executed by the server
  • Requires server interaction
  • Increases server processing
  • Main language: PHP
  • Code is hidden from users

⚖️ Comparison

Feature Client-side Server-side
Where it runs Browser Server
Speed Faster Slower
Processing Low server load High server load
Security Less secure More secure
Code visibility Visible Hidden

🧠 Quick Memory

  • Client → browser
  • Server → server
  • Client → JavaScript
  • Server → PHP
