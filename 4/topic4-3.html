🔄 Dataflow Diagrams (DFD)

📌 What is a Dataflow Diagram?

A dataflow diagram (DFD) shows how data moves through a system, including input, processing, storage, and output.

Purpose

Shows how data flows
  • Input of data into system
  • Processing of data
  • Storage of data
  • Output of data
💡 Exam Tip: DFD = movement of data, not steps

🔣 DFD Symbols

1

📥 Data Source / Sink

Where data enters or leaves the system (e.g. user, scanner).

2

⚙️ Process

Action performed on data (e.g. scanning, calculating).

3

💾 Data Store

Where data is stored (e.g. database).

4

➡️ Data Flow

Arrow showing direction of data movement.

💡 Exam Tip: Always label arrows with data name

✈️ Example: Airport Check-in System

1

📥 Data Sources

  • Customer
  • Barcode scanner
  • Weighing scales
2

⚙️ Processes

  • Scan boarding pass
  • Weigh luggage
  • Print luggage label
3

💾 Data Stores

  • Customer database
  • Luggage database

🧠 Summary

A DFD helps visualise how data moves through a system by showing sources, processes, storage, and outputs.

💡 Exam Tip: DFD focuses on data flow, not control flow
