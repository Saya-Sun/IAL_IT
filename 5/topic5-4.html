🗄️ Structured Data in a Database

Database Table

Data organised in rows and columns
  • Field: column (one type of data)
  • Record: row (one item)
💡 Exam Tip: Field = column, Record = row

📊 Table (Field & Record)

Song title Artist Genre
Happy in love T. Singer Pop
Dance all day The Dances Dance
Rock and rock The 2 Rs Rock
Let's pop! T. Singer Pop

Field: Song title, Artist, Genre (columns)
Record: One row of data

  • e.g. "Happy in love", "T. Singer", "Pop"

💡 Exam Tip: Field = column, Record = row

Why is a Primary Key Needed?

Ensures each record is unique
  • Prevents duplicate records in a table
  • Allows a specific record to be found quickly
  • Makes updating and deleting data accurate
  • Used to link tables together (relationships)
💡 Exam Tip: Primary key = unique + identify + link tables

Primary Key

Unique identifier
  • Each record has a unique value
  • No duplicates allowed
  • Example: Song ID
✅ Benefits
  • Prevents duplicate records
  • Helps update/delete data
  • Links tables together
2

Foreign Key

Links tables

A primary key from one table used in another table to create a relationship.

💡 Exam Tip: Foreign key = link between tables
3

Composite Key

Combination of fields

Two or more fields combined to create a unique identifier.

Example: Song Title + Artist
