🔗 Relationships Between Entities

📌 What is an Entity?

An entity is another name for a table in a database. Relationships link entities together.

1

One-to-One

1 record → 1 record

Each record in one table is linked to only one record in another table.

Example: One student → one emergency contact

student
emergency contact
💡 Created using primary key → foreign key
2

One-to-Many

1 record → many records

One record in a table is linked to multiple records in another table.

Example: One student → many classes

  • Student ID → appears multiple times in class records
Student
Classes
💡 Most common relationship in databases
3

Many-to-Many

Many records ↔ many records

Multiple records in one table relate to multiple records in another.

Example: Customers ↔ Products

Customers
Products
⚠️ Not used directly

This is broken into two one-to-many relationships using a joining table.

Joining Entity

Resolves many-to-many

A third table is created to link the two entities.

  • Contains primary key
  • Includes foreign keys from both tables

Example: Orders table linking Customers and Products

💡 Exam Tip: Many-to-many must be split into two one-to-many

🧠 Summary

  • One-to-one → 1 to 1
  • One-to-many → 1 to many
  • Many-to-many → needs joining table
